## Compact bootstrap architecture

The fixed file inputs are exactly:

1. the one applicable harness wrapper, `AGENTS.md` or `CLAUDE.md`;
2. `docs/governance/AGENT_INSTRUCTIONS.md`;
3. `docs/governance/current-state.json`;
4. `AGENT_HANDOFF.md`.

Live Git and environment preflight occurs before those reads. Task-specific
plans, specifications, contracts, traceability, rollback evidence, tests, and
bounded ledger entries are followed only through explicit pointers and are
excluded from the fixed cost.

`AGENT_INSTRUCTIONS.md` owns active operational governance.
`current-state.json` owns only current operational continuity.
`AGENT_HANDOFF.md` is generated and owns no independent fact.

product, and technical semantics. Git, code, migrations, tests, and append-only
ledgers retain their evidence roles.
